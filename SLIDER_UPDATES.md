# Оновлення слайдера - Заокруглені краї та кнопки навігації

## 📋 Що було зроблено:

### 1. **Заокруглені краї (Rounded corners)**
- ✅ Додано `borderRadius: 16` до контейнера слайдера
- ✅ Зображення слайдів тепер мають заокруглені краї
- ✅ Нижній оверлей з назвою також має заокруглені нижні краї
- ✅ Додано відступи по боках (`paddingHorizontal: 16`)

### 2. **Кнопки навігації "ліворуч" / "праворуч"**
- ✅ Додано білі круглі кнопки з обох боків слайдера
- ✅ Кнопки мають символи стрілок `‹` та `›`
- ✅ Додано тінь (shadow) для кнопок
- ✅ Напівпрозорий білий фон `rgba(255, 255, 255, 0.9)`
- ✅ Кнопки розміщені по центру висоти слайдера

### 3. **Покращена пагінація**
- ✅ Білі точки замість зелених
- ✅ Активна точка тепер більша (10x10px замість 8x8px)
- ✅ Напівпрозорі неактивні точки `rgba(255, 255, 255, 0.4)`

### 4. **Покращене прокручування**
- ✅ Використання `snapToInterval` для плавного прокручування
- ✅ `decelerationRate="fast"` для швидкої зупинки
- ✅ Правильний розрахунок активного індексу з урахуванням нової ширини

## 🔧 Технічні деталі:

### Стилі кнопок:
- Розмір: `40x40px`
- Border radius: `20px` (повний круг)
- Фон: `rgba(255, 255, 255, 0.9)`
- Позиція: абсолютна, по центру вертикально
- Відступи від країв: `16px`

### Розміри слайдера:
- Висота: `190px`
- Ширина: `100% екрану - 32px` (по 16px з кожного боку)
- Border radius: `16px`

### Функціонал:
- Кнопки `goToPrevious()` та `goToNext()` для ручної навігації
- Циклічне прокручування (останній слайд → перший)
- Автоплей продовжує працювати (3 секунди)
- Зупинка автоплею при ручному прокручуванні

## 📝 Використання:

Компонент працює так само, як і раніше:

```tsx
<PromoSlider
  data={slidesData}
  onSlidePress={handleSlidePress}
/>
```

Ніяких змін в API компонента не потрібно!
