# Очистка стора при виході з системи

## Що було зроблено

✅ Оновлено функцію `logout()` в `authSlice.ts`

✅ При виході з системи тепер очищаються:
  - Дані користувача (`user`)
  - Список обраних товарів (`favorites`)
  - Кошик (`cart`)
  - Статус авторизації (`isAuthenticated`)

## Файли, які були змінені

### src/state/slices/authSlice.ts

Оновлено функцію `logout()`:

```typescript
logout: () => {
  set((state) => ({
    user: null,
    favorites: [], // Очищаємо обране
    cart: [], // Очищаємо кошик
    settings: { ...state.settings, isAuthenticated: false },
  }));
},
```

## Як це працює

1. Користувач натискає кнопку "Вийти" в бічному меню
2. Викликається функція `handleLogout()` в `SideMenu` компоненті
3. Функція `logout()` зі стору очищає:
   - Інформацію про користувача
   - Всі обрані товари
   - Всі товари з кошика
   - Встановлює `isAuthenticated = false`
4. Користувач перенаправляється на головний екран
5. Бічне меню закривається

## Переваги

✨ **Безпека**: Особисті дані користувача не залишаються в застосунку після виходу

✨ **Приватність**: Обране та кошик очищаються, щоб інший користувач не бачив ваших товарів

✨ **Чистий старт**: Кожен новий вхід починається з чистого стану

## Зберігання даних

Важливо розуміти:
- Дані зберігаються в AsyncStorage через Zustand persist middleware
- При виході дані очищаються як з пам'яті (state), так і з AsyncStorage
- При наступному вході користувача wishlist та cart завантажаться з сервера

## Тестування

Щоб перевірити, чи працює очистка:

1. Увійдіть в систему
2. Додайте кілька товарів в обране
3. Додайте кілька товарів в кошик
4. Натисніть "Вийти" в бічному меню
5. Перевірте, що:
   - Обране порожнє
   - Кошик порожній
   - В бічному меню показується кнопка "Увійти" замість даних користувача

## Примітки

- Категорії (`categories`) НЕ очищаються при виході, бо вони загальні для всіх
- Налаштування (`settings`) зберігаються (тема, мова тощо)
- Доступні мови (`availableLanguages`) зберігаються
